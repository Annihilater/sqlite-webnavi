# SQLite Webnavi - 网址导航系统

一个基于 Express.js 和 SQLite 的网址导航系统，提供响应式的网址卡片展示、分类筛选和搜索功能。

## 功能特性

### 前台展示
- 响应式网址卡片展示
- 网站分类筛选
- 标签筛选
- 实时搜索功能
- 支持新窗口/当前窗口打开链接
- 新品和热门标记

### 后台管理
- 用户认证系统
- 网址管理（增删改查）
- 分类管理
- 标签管理
- 图片上传（支持文件上传、URL导入、截图粘贴）
- 拖拽排序

## 技术栈

### 前端
- HTML5
- Tailwind CSS
- Font Awesome
- Google Fonts
- 原生 JavaScript

### 后端
- Express.js
- SQLite 数据库
- JWT 用户认证

## 安装

1. 克隆项目
```bash
git clone [项目地址]
cd sqlite-webnavi
```

2. 安装依赖
```bash
npm install
```

3. 启动服务器
```bash
npm start
```

4. 访问系统
- 前台：http://localhost:3000
- 后台：http://localhost:3000/admin
- 默认管理员账号：admin
- 默认管理员密码：admin

## 开发说明

### 数据库设计
系统使用 SQLite 数据库，包含以下主要表：
- sites：网站信息表
- categories：分类表
- tags：标签表
- site_categories：网站-分类关联表
- site_tags：网站-标签关联表
- users：用户表

### API 接口
系统提供完整的 RESTful API：
- 认证接口：/api/auth/*
- 网站接口：/api/sites/*
- 分类接口：/api/categories/*
- 标签接口：/api/tags/*
- 上传接口：/api/upload/*

### 系统改进

#### 首页动态数据获取
首页现在通过API从后端获取分类和网站数据，而不是使用硬编码的静态内容。这带来以下好处：
- 添加新网站或分类时无需修改前端代码
- 管理员在后台的更改立即反映在前端
- 更好的性能和可维护性

实现方式：
1. 页面加载时通过 `fetch('/api/sites')` 和 `fetch('/api/categories')` 获取数据
2. 动态创建分类选项卡和网站卡片
3. 保留原有的搜索和筛选功能，但针对动态内容进行了优化

#### 用户体验改进
为提升用户体验，进行了以下优化：

1. **优化加载状态**：
   - 添加精美的加载动画，提供视觉反馈
   - 错误处理更加友好，显示明确的错误信息和重试按钮

2. **智能占位图像**：
   - 为没有截图的网站自动生成基于名称的占位图像
   - 占位图使用从网站名称生成的独特颜色，并显示网站首字母
   - 图片加载失败时自动回退到占位图，确保界面美观

3. **性能优化**：
   - 并行加载API数据，减少等待时间
   - 优化卡片动画，使页面加载更加流畅
   - 使用数据缓存减少不必要的API调用 

- **标签关联问题修复**: 修复了在添加或编辑网站时，标签无法正确关联的问题。现在添加的标签会立即与网站关联，无需二次编辑。
- **界面优化**: 提升了标签选择和管理的用户体验，增加了更多的日志记录以便于调试。
- **数据加载优化**: 改进了前端数据加载逻辑，提高了页面响应速度。
